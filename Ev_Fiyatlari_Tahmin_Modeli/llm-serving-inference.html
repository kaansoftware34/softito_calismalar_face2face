Serving Stack Nedir?

Model Serving

Eğitilmiş bir modeli HTTP/gRPC endpoint olarak dışarıya açma sürecidir. İstemciler bu endpoint'e istek gönderir, model çıkarım (inference) yapar, sonucu döner.

Inference Engine

Modeli GPU/CPU'ya yükleyip çalıştıran çekirdek bileşen. Batching, quantization ve bellek yönetimi gibi optimizasyonları burada yapılır.

LLM-Specific Challenges

LLM'ler uzun context, token-by-token üretim (autoregressive decoding) ve KV-cache yönetimi gibi kendine özgü zorluklar sunar. Klasik serving frameworkleri bu konularda yetersiz kalır.

Kubernetes Entegrasyonu

K8s üzerinde model serving; GPU node scheduling, autoscaling, health check ve rolling update gibi platform özelliklerinden yararlanmayı sağlar.

01 · Platform

KServe

KServe

Kubernetes-native Model Serving

Kubernetes CRD'leri ile model deploy et, autoscale et, canary release yap.

KServe (eski adıyla KFServing), Kubernetes üzerinde model servis etmek için CNCF ekosisteminden gelen açık kaynak bir platformdur. InferenceService adı verilen custom resource ile model deployment, autoscaling ve traffic splitting tek bir YAML ile yönetilir.

Temel Özellikler

  • InferenceService CRD ile declarative deployment
  • KNative üzerinden scale-to-zero desteği
  • Canary / Blue-green traffic splitting
  • Sklearn, XGBoost, PyTorch, TF, ONNX, HuggingFace desteği
  • Model explainability (Alibi) entegrasyonu
  • Pre/post processing transformer pod'ları

Mimari Katmanlar

  • Predictor — modeli çalıştıran asıl pod
  • Transformer — istek/yanıt dönüşümü
  • Explainer — tahmin açıklaması
  • ModelMesh — yoğun model senaryoları için high-density serving
  • Istio ile mTLS ve traffic management
llama-inference-service.yaml YAML 
apiVersion: serving.kserve.io/v1beta1
kind: InferenceService
metadata:
  name: llama-3-8b
  namespace: ml-serving
spec:
  predictor:
    model:
      modelFormat:
        name: huggingface
      storageUri: s3://my-bucket/llama-3-8b
      resources:
        limits:
          nvidia.com/gpu: "1"
          memory: 24Gi
        requests:
          nvidia.com/gpu: "1"
          memory: 20Gi
  transformer:
    containers:
      - name: custom-transformer
        image: my-registry/chat-transformer:v1
inference-request.py Python 
import requests

# KServe endpoint'e istek gönder
url = "http://llama-3-8b.ml-serving.svc.cluster.local/v1/models/llama-3-8b:predict"

payload = {
    "instances": [{
        "prompt": "Kubernetes nedir?",
        "max_new_tokens": 256,
        "temperature": 0.7
    }]
}

response = requests.post(url, json=payload)
print(response.json()["predictions"][0])

02 · Inference Server

Triton Inference Server

Triton

NVIDIA Triton Inference Server

Multi-framework, high-throughput, production-grade inference.

NVIDIA'nın geliştirdiği Triton, TensorRT, PyTorch, ONNX, TensorFlow ve OpenVINO modellerini aynı anda servis edebilen yüksek performanslı bir inference sunucusudur. Dynamic batching ve concurrent model execution gibi özelliklerle GPU kullanımını maksimize eder.

Multi

Desteklenen Backend'ler

  • TensorRT — NVIDIA GPU optimizasyonu
  • PyTorch (TorchScript) — dinamik graflar
  • ONNX Runtime — cross-platform
  • TensorFlow SavedModel / GraphDef
  • OpenVINO — Intel CPU/iGPU
  • Python Backend — custom inference kodu

Performans Özellikleri

  • Dynamic Batching — istekleri otomatik grupla
  • Concurrent Model Execution — aynı anda N kopya
  • Model Ensemble — zincirleme pipeline
  • CUDA Shared Memory — sıfır kopya transfer
  • Rate Limiter — öncelikli inference sırası

03 · LLM Engine

vLLM

vLLM

High-Throughput LLM Serving

PagedAttention ile KV-cache yönetimi; OpenAI API uyumlu endpoint.

vLLM, UC Berkeley tarafından geliştirilen ve LLM servisi için özelleşmiş bir inference engine'dir. Temel yeniliği olan PagedAttention, GPU belleğini sanal bellek mantığıyla yönetir; bu sayede KV-cache parçalanmasını azaltır ve throughput'u dramatik biçimde artırır.

PagedAttention Mantığı

# Klasik Serving — KV-cache parçalanması
Request A: ████████████░░░░ ← 25% wasted
Request B: ██████░░░░░░░░░░ ← 62% wasted

# vLLM PagedAttention — sanal blok yönetimi
Physical Memory: ████ ████ ████ ████
↑ ↑ ↑ ↑
Block Table: A[0] B[0] A[1] B[1] ← interleaved, no waste

Temel Özellikler

  • PagedAttention — blok tabanlı KV-cache
  • Continuous Batching — istekleri sürekli birleştir
  • Tensor Parallelism — çoklu GPU dağıtımı
  • Speculative Decoding — taslak model ile hızlanma
  • AWQ / GPTQ / SqueezeLLM quantization
  • LoRA / QLoRA adapter yükleme (hot-swap)

Desteklenen Modeller

  • Llama 3 / Llama 2 ailesi
  • Mistral / Mixtral MoE
  • Qwen 2.5, DeepSeek, Gemma
  • Falcon, MPT, StarCoder
  • Phi-3, Command-R+
  • Tüm HuggingFace AutoModelForCausalLM modelleri

vllm-client.py Python — OpenAI SDK ile 
from openai import OpenAI

# vLLM, OpenAI API'si ile birebir uyumludur
client = OpenAI(
    api_key="not-needed",
    base_url="http://vllm-service:8000/v1"
)

# Streaming ile yanıt al
stream = client.chat.completions.create(
    model="meta-llama/Meta-Llama-3-8B-Instruct",
    messages=[
        {"role": "user", "content": "Kubernetes pod nedir?"}
    ],
    max_tokens=512,
    temperature=0.7,
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
04 · Local & Cluster

Ollama

Ollama

Local Model Runtime

Laptop'tan cluster'a — tek komutla model çalıştır.

Ollama, LLM'leri yerel ortamlarda ve küçük cluster'larda kolayca çalıştırmak için tasarlanmış hafif bir runtime'dır. llama.cpp tabanlı olup CPU'da da çalışabilir; geliştirme ortamı veya edge deployment için idealdir.

Güçlü Yönleri

  • Sıfır konfigürasyon ile model indirme ve çalıştırma
  • CPU + GPU hybrid çalışma (Metal, CUDA, ROCm)
  • Modelfile ile özel model tanımları
  • OpenAI uyumlu REST API
  • Çok küçük memory footprint
  • Kubernetes'te DaemonSet olarak node başına deploy

Sınırlamaları

  • Tensor parallelism desteği kısıtlı
  • Yüksek concurrent istek kapasitesi düşük
  • vLLM kadar throughput optimizasyonu yok
  • Production yük için tek başına yetmez
  • Model registry yönetimi basit kalır
